# 기여 가이드 (Contributing Guide)

뉴스 큐레이션 프로젝트에 관심을 가져주셔서 감사합니다! 여러분의 기여는 프로젝트를 더 발전시키는 데 큰 도움이 됩니다.

## 🛠️ 개발 환경 설정

1. **Node.js**: LTS 버전(v18 이상) 사용을 권장합니다.
2. **패키지 매니저**: `npm`을 사용합니다.
3. **데이터베이스**: 로컬 개발 환경에서는 SQLite를 사용하므로 별도의 DB 설치가 필요 없습니다.

## 💻 코드 스타일

- **TypeScript**: 엄격한 타입 체크(`strict: true`)를 준수해주세요.
- **Linting**: 커밋 전 `npm run lint`를 실행하여 코드 스타일을 확인해주세요.
- **Formatting**: 프로젝트의 Prettier 설정을 따릅니다.

## 🧪 테스트

새로운 기능을 추가하거나 버그를 수정을 할 때는 테스트를 작성하는 것을 권장합니다.

```bash
# 전체 테스트 실행
npm test

# 감시 모드로 테스트 실행
npm run test:watch
```

## 🔀 풀 리퀘스트 (Pull Request) 절차

1. Issue를 생성하여 변경 사항에 대해 논의합니다.
2. `main` 브랜치에서 새로운 기능 브랜치(`feature/feature-name`)를 생성합니다.
3. 코드를 작성하고 커밋합니다. 커밋 메시지는 명확하게 작성해주세요.
4. 원격 저장소에 브랜치를 푸시하고 PR을 생성합니다.
5. 코드 리뷰를 받고, 필요한 수정을 진행합니다.
6. 리뷰가 완료되면 `main` 브랜치에 머지됩니다.

## 🐛 버그 신고

버그를 발견하시면 Issue 탭에 자세한 내용(재현 방법, 예상 동작, 실제 동작 등)을 남겨주세요.
